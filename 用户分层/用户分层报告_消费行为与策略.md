# 消费行为分析与策略建议

## 1. 消费频次分布分析

### 1.1 频次分层定义

基于月度购买次数，将用户分为四个消费频次层级：

| 频次层级 | 定义 | 典型特征 |
|---------|------|----------|
| 单次用户 | 仅下单1次 | 体验型用户，转化风险高 |
| 低频用户 | 2-3单/月 | 偶尔消费，习惯未养成 |
| 中频用户 | 4-8单/月 | 稳定消费，每周1-2次 |
| 高频用户 | 9+单/月 | 忠诚用户，几乎每天消费 |

### 1.2 频次分布SQL查询

```sql
-- 消费频次分布分析（月度）
WITH user_monthly_orders AS (
    SELECT
        user_no,
        DATE_FORMAT(DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')), '%Y-%m') AS order_month,
        COUNT(DISTINCT id) AS order_cnt
    FROM ods_luckyus_sales_order.v_order
    WHERE status = 90
      AND INSTR(tenant, 'IQ') = 0
      AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
      AND DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York'))
          BETWEEN '2026-01-01' AND '2026-01-31'
    GROUP BY user_no, order_month
)
SELECT
    CASE
        WHEN order_cnt = 1 THEN '单次用户'
        WHEN order_cnt BETWEEN 2 AND 3 THEN '低频用户'
        WHEN order_cnt BETWEEN 4 AND 8 THEN '中频用户'
        WHEN order_cnt >= 9 THEN '高频用户'
    END AS frequency_tier,
    COUNT(DISTINCT user_no) AS user_count,
    ROUND(COUNT(DISTINCT user_no) * 100.0 / SUM(COUNT(DISTINCT user_no)) OVER(), 2) AS user_pct,
    ROUND(AVG(order_cnt), 2) AS avg_orders,
    MIN(order_cnt) AS min_orders,
    MAX(order_cnt) AS max_orders
FROM user_monthly_orders
GROUP BY frequency_tier
ORDER BY
    CASE frequency_tier
        WHEN '单次用户' THEN 1
        WHEN '低频用户' THEN 2
        WHEN '中频用户' THEN 3
        WHEN '高频用户' THEN 4
    END;
```

### 1.3 各群体特征与价值

**典型分布估算**（基于茶饮行业基准）：

- **单次用户（~40-50%）**：新客流失高危群体，需首单转化策略
- **低频用户（~25-30%）**：习惯培养期，需频次提升激励
- **中频用户（~15-20%）**：稳定贡献层，需价值深挖
- **高频用户（~5-10%）**：核心忠诚用户，需VIP维护

**运营重点**：
- 单次→低频：48小时黄金窗口期，复购券促转化
- 低频→中频：周度满减活动，培养购买习惯
- 中频→高频：会员积分体系，增强粘性
- 高频维护：专属权益，防止流失

## 2. 客单价分层分析

### 2.1 客单价分层定义

根据门店定价区间和用户消费水平划分：

| 客单价层级 | 定义 | 用户画像 |
|-----------|------|----------|
| 低客单 | <$5 | 价格敏感型，基础款消费 |
| 中客单 | $5-$8 | 主流消费群体，标准产品 |
| 高客单 | $8+ | 品质追求型，加料/大杯 |

### 2.2 客单价分布SQL查询

```sql
-- 客单价分层分析
WITH user_avg_spending AS (
    SELECT
        user_no,
        COUNT(DISTINCT id) AS total_orders,
        SUM(pay_money) AS total_spending,
        ROUND(SUM(pay_money) / COUNT(DISTINCT id), 2) AS avg_order_value
    FROM ods_luckyus_sales_order.v_order
    WHERE status = 90
      AND INSTR(tenant, 'IQ') = 0
      AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
      AND DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York'))
          BETWEEN '2026-01-01' AND '2026-01-31'
    GROUP BY user_no
)
SELECT
    CASE
        WHEN avg_order_value < 5 THEN '低客单(<$5)'
        WHEN avg_order_value BETWEEN 5 AND 8 THEN '中客单($5-$8)'
        WHEN avg_order_value > 8 THEN '高客单($8+)'
    END AS aov_tier,
    COUNT(DISTINCT user_no) AS user_count,
    ROUND(COUNT(DISTINCT user_no) * 100.0 / SUM(COUNT(DISTINCT user_no)) OVER(), 2) AS user_pct,
    ROUND(AVG(avg_order_value), 2) AS avg_aov,
    ROUND(AVG(total_orders), 2) AS avg_order_frequency,
    ROUND(SUM(total_spending) / COUNT(DISTINCT user_no), 2) AS avg_ltv_per_user
FROM user_avg_spending
GROUP BY aov_tier
ORDER BY
    CASE aov_tier
        WHEN '低客单(<$5)' THEN 1
        WHEN '中客单($5-$8)' THEN 2
        WHEN '高客单($8+)' THEN 3
    END;
```

### 2.3 客单价与复购的关系

```sql
-- 客单价与复购率交叉分析
WITH user_metrics AS (
    SELECT
        user_no,
        COUNT(DISTINCT id) AS order_cnt,
        ROUND(SUM(pay_money) / COUNT(DISTINCT id), 2) AS avg_order_value
    FROM ods_luckyus_sales_order.v_order
    WHERE status = 90
      AND INSTR(tenant, 'IQ') = 0
      AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
      AND DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York'))
          BETWEEN '2026-01-01' AND '2026-01-31'
    GROUP BY user_no
)
SELECT
    CASE
        WHEN avg_order_value < 5 THEN '低客单'
        WHEN avg_order_value BETWEEN 5 AND 8 THEN '中客单'
        ELSE '高客单'
    END AS aov_tier,
    CASE
        WHEN order_cnt = 1 THEN '单次'
        WHEN order_cnt BETWEEN 2 AND 3 THEN '低频'
        WHEN order_cnt BETWEEN 4 AND 8 THEN '中频'
        ELSE '高频'
    END AS freq_tier,
    COUNT(DISTINCT user_no) AS user_count,
    ROUND(AVG(avg_order_value), 2) AS avg_aov,
    ROUND(AVG(order_cnt), 2) AS avg_frequency
FROM user_metrics
GROUP BY aov_tier, freq_tier
ORDER BY
    CASE aov_tier WHEN '低客单' THEN 1 WHEN '中客单' THEN 2 ELSE 3 END,
    CASE freq_tier WHEN '单次' THEN 1 WHEN '低频' THEN 2 WHEN '中频' THEN 3 ELSE 4 END;
```

**关键洞察**：
- 中客单用户复购率通常最高（价格与价值平衡）
- 低客单用户需促销刺激，但长期价值有限
- 高客单用户虽频次较低，但LTV贡献显著

### 3.2 品类偏好SQL查询

```sql
-- 用户品类偏好分析（基于门店销售汇总表）
WITH user_category_preference AS (
    SELECT
        dt,
        shop_name,
        one_category_name,

)

    one_category_name AS category,
    SUM(total_cups) AS total_cups,

    SUM(total_orders) AS total_orders,
    ROUND(SUM(total_revenue), 2) AS total_revenue,
    ROUND(SUM(total_revenue) / SUM(total_cups), 2) AS avg_price_per_cup

ORDER BY total_cups DESC;
```

## 4. 时段消费分析

### 4.1 消费时段划分

| 时段 | 时间范围 | 消费动机 |
|------|---------|----------|
| 早高峰 | 7:00-10:00 | 上班路上，提神刚需 |
| 午间 | 11:00-14:00 | 午餐后提神，下午续航 |
| 下午茶 | 14:00-17:00 | 休闲放松，社交场景 |
| 晚间 | 17:00-22:00 | 加班续杯，轻度社交 |

### 4.2 时段消费分布SQL查询

```sql
-- 消费时段分布分析
WITH order_by_hour AS (
    SELECT
        user_no,
        DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')) AS order_date,
        HOUR(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')) AS order_hour,
        COUNT(DISTINCT id) AS order_cnt,
        SUM(pay_money) AS total_spending
    FROM ods_luckyus_sales_order.v_order
    WHERE status = 90
      AND INSTR(tenant, 'IQ') = 0
      AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
      AND DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York'))
          BETWEEN '2026-01-01' AND '2026-01-31'
    GROUP BY user_no, order_date, order_hour
)
SELECT
    CASE
        WHEN order_hour BETWEEN 7 AND 9 THEN '早高峰(7-10am)'
        WHEN order_hour BETWEEN 10 AND 13 THEN '午间(10am-2pm)'
        WHEN order_hour BETWEEN 14 AND 16 THEN '下午茶(2-5pm)'
        WHEN order_hour BETWEEN 17 AND 21 THEN '晚间(5-10pm)'
        ELSE '其他时段'
    END AS time_period,
    COUNT(DISTINCT user_no) AS unique_users,
    SUM(order_cnt) AS total_orders,
    ROUND(SUM(order_cnt) * 100.0 / SUM(SUM(order_cnt)) OVER(), 2) AS order_pct,
    ROUND(SUM(total_spending), 2) AS total_revenue,
    ROUND(SUM(total_spending) / SUM(order_cnt), 2) AS avg_order_value
FROM order_by_hour
GROUP BY time_period
ORDER BY
    CASE time_period
        WHEN '早高峰(7-10am)' THEN 1
        WHEN '午间(10am-2pm)' THEN 2
        WHEN '下午茶(2-5pm)' THEN 3
        WHEN '晚间(5-10pm)' THEN 4
        ELSE 5
    END;
```

### 4.3 时段消费用户画像

```sql
-- 识别主要消费时段偏好用户
WITH user_time_preference AS (
    SELECT
        user_no,
        CASE
            WHEN HOUR(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')) BETWEEN 7 AND 9 THEN '早高峰'
            WHEN HOUR(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')) BETWEEN 10 AND 13 THEN '午间'
            WHEN HOUR(CONVERT_TZ(create_time, @@time_zone, 'America/New_York')) BETWEEN 14 AND 16 THEN '下午茶'
            ELSE '晚间'
        END AS time_period,
        COUNT(DISTINCT id) AS order_cnt
    FROM ods_luckyus_sales_order.v_order
    WHERE status = 90
      AND INSTR(tenant, 'IQ') = 0
      AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
      AND DATE(CONVERT_TZ(create_time, @@time_zone, 'America/New_York'))
          BETWEEN '2026-01-01' AND '2026-01-31'
    GROUP BY user_no, time_period
),
user_primary_time AS (
    SELECT
        user_no,
        time_period,
        order_cnt,
        ROW_NUMBER() OVER (PARTITION BY user_no ORDER BY order_cnt DESC) AS rn
    FROM user_time_preference
)
SELECT
    time_period AS primary_time_period,
    COUNT(DISTINCT user_no) AS user_count,
    ROUND(AVG(order_cnt), 2) AS avg_orders_in_period
FROM user_primary_time AS (
    SELECT * FROM user_primary_time WHERE rn = 1
)
GROUP BY time_period
ORDER BY user_count DESC;
```

**运营策略**：
- **早高峰用户**：提前下单功能，到店即取，提升效率
- **午间用户**：午餐套餐组合，饮品+轻食搭配
- **下午茶用户**：限时折扣活动（2-5pm专享），社交分享奖励
- **晚间用户**：夜间续杯卡，低因咖啡推荐

## 5. 门店区域差异分析

### 5.1 门店销售表现对比


### 5.2 区域消费特征差异

根据门店位置（商圈、校园、社区）分析消费行为差异：

**商圈门店特征**：
- 客流高，周末高峰明显
- 客单价较高，特调饮品占比大
- 下午茶和晚间时段表现强

**校园门店特征**：
- 早高峰和下午茶时段集中
- 价格敏感，促销活动效果明显
- 学期初和期末消费波动大

**社区门店特征**：
- 用户忠诚度高，复购稳定
- 早高峰为主，工作日强于周末
- 客单价中等，经典咖啡为主

## 6. 综合运营策略矩阵

### 6.1 用户价值分层与策略对应

基于消费频次、客单价、品类偏好三维度交叉分析，制定精准运营策略：

| 用户类型 | 消费特征 | 核心策略 | 具体动作 |
|---------|---------|---------|---------|
| **高价值用户** | 高频+高客单 | VIP权益维护 | ① 会员积分加速（2倍积分）<br>② 生日专属折扣<br>③ 新品优先试饮权<br>④ 专属客服通道 |
| **成长期用户** | 中频+中客单 | 频次提升+客单价拉升 | ① 周末双倍积分活动<br>② 升杯/加料半价券<br>③ 品类交叉推荐<br>④ 满4杯送1杯活动 |
| **沉睡用户** | 30天+未消费 | 召回激活 | ① 48小时限时返场券（7折）<br>② 推送"好久不见"关怀消息<br>③ 赠送小额无门槛券<br>④ 新品推荐吸引回流 |

### 6.2 优惠券策略设计

**频次提升型券包**：
```
目标：单次用户 → 低频用户
券包：第2单7折 + 第3单买一送一
有效期：首购后15天内
预期效果：复购率提升20-30%
```

**客单价拉升型券包**：
```
目标：低客单用户 → 中客单用户
券包：满$8减$2 + 升杯券（+$1升大杯）
有效期：30天
预期效果：客单价提升15-20%
```

**留存激活型券包**：
```
目标：沉睡用户召回
券包：无门槛$3券 + 满$10减$3
有效期：7天
预期效果：召回率15-20%
```

## 7. 数据监控与迭代优化

### 7.1 核心监控指标

**日常监控看板**：

```sql
-- 每日核心指标监控（使用汇总表）
SELECT
    dt AS 日期,
    COUNT(DISTINCT shop_name) AS 营业门店数,
    SUM(sku_cnt) AS 总杯量,
    SUM(order_cnt) AS 总订单数,
    ROUND(SUM(pay_amount), 2) AS 总销售额,
    ROUND(SUM(pay_amount) / SUM(order_cnt), 2) AS 客单价,
    ROUND(SUM(sku_cnt) / SUM(order_cnt), 2) AS 单均杯数
FROM dw_ads.ads_mg_sku_shop_sales_statistic_d_1d
WHERE dt >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
  AND tenant = 'LKUS'
  AND one_category_name = 'Drink'
  AND shop_name NOT IN ('NJ Test Kitchen', 'NJ Test Kitchen 2')
GROUP BY dt
ORDER BY dt DESC;
```

**周度复购监控**：

```sql
-- 周复购率趋势（使用复购汇总表，秒出）
SELECT
    src_week_first_date AS 起始周,
    dst_week_first_date AS 目标周,
    src_usr_cnt AS 起始用户数,
    dst_usr_cnt AS 复购用户数,
    ROUND(dst_usr_cnt * 100.0 / src_usr_cnt, 2) AS 复购率,
    ROUND(dst_drink_cnt / dst_usr_cnt, 2) AS 人均复购杯数
FROM dw_ads.ads_user_order_rep_info_d_nw
WHERE src_week_first_date >= DATE_SUB(CURDATE(), INTERVAL 8 WEEK)
  AND DATEDIFF(dst_week_first_date, src_week_first_date) = 7
ORDER BY src_week_first_date DESC;
```

### 7.2 策略效果评估

**券包核销率监控**：
- 核销率 < 30%：券面吸引力不足或推送人群不精准
- 核销率 30-60%：正常水平，持续优化
- 核销率 > 60%：高质量券包，可复制推广

**ROI计算公式**：
```
营销ROI = (活动期间增量GMV - 券补贴成本) / 券补贴成本
目标ROI：≥3（即每投入$1券补贴带来$3以上增量销售）
```

### 7.3 迭代优化建议

**AB实验框架**：
1. **优惠券面值测试**：$2 vs $3 vs 7折券，找到最优性价比
2. **推送文案测试**：理性诉求 vs 情感诉求，提升点击率
3. **会员权益测试**：积分倍数 vs 实物礼品，优化留存效果
4. **推送时机测试**：早8点 vs 午12点 vs 晚6点，找到最佳触达时机

**数据驱动优化循环**：
```
数据监控 → 发现问题 → 假设验证（AB实验）→ 策略迭代 → 效果评估 → 下一轮优化
```
