# 工作计划策略日志中心

这个项目用于沉淀你所有运营动作，支持两件事：
- 日常管理：随时查看当前进展、已生效策略、下周计划
- 分析联动：为数据分析提供统一上下文输入，减少反复对齐

## 目录说明
- `00_项目总览.md`：项目目标、范围、当前优先级
- `01_策略台账.csv`：策略级结构化记录（便于筛选和分析）
- `02_数据分析上下文.md`：给数据分析同学/AI的标准输入
- `03_动作日志/`：按月记录每次动作、调整、复盘
- `07_工作待办.md`：日常执行待办与周期任务清单
- `06_实验配置存档/`：关键实验后台配置截图信息存档

## 维护规则
1. 新策略先在 `01_策略台账.csv` 建一条记录。
2. 每次上线、调整、暂停都写入 `03_动作日志/`。
3. 周会前更新 `02_数据分析上下文.md` 的“本周重点问题”。
4. 所有时间统一使用：`America/New_York`（美东）。
5. 若出现时间冲突或相似项目名冲突，先记录到 `99_待确认冲突.md`，经你确认后再更新主档。

## 快速维护命令
- 查看月度日志：`./工作计划_策略日志中心/scripts/workplan.sh month-view 2026-03`
- 新增动作日志：`./工作计划_策略日志中心/scripts/workplan.sh add-log "项目" "动作" "状态" "目标指标" "备注"`
- 新增策略台账：`./工作计划_策略日志中心/scripts/workplan.sh add-strategy "strategy_id" "strategy_name" "goal" "audience" "trigger_channel" "offer_or_action" "ab_design" "core_metrics" "secondary_metrics" "status" "start_date" "owner" "notes"`
- `04_协作使用说明.md`：三个核心使用场景（今日待办/动作记录/周报）
- `05_动作录入模板.md`：给我的标准输入模板
- `99_待确认冲突.md`：时间冲突、项目命名冲突的待确认清单
