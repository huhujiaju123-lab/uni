### 1.2 覆盖面评估

```
全类型刚需（所有用户都要）：
  ✅ 一句话概括 + 话题标签 + 听前预览
  → 这是产品的基础层，必须做好

高信息密度类刚需（知识+访谈+商业，占小宇宙 60%+）：
  ✅ 章节结构 + 核心观点 + 思维导图 + 内联图表
  → 当前产品已有，需要优化

类型特有需求（覆盖面窄但满足感强）：
  ⚡ 故事型 → 情节时间线 + 人物关系
  ⚡ 访谈型 → 嘉宾信息卡 + 对话流图
  ⚡ 闲聊型 → 社交分享卡片

差异化机会（竞品薄弱）：
  🔵 社交分享卡片/海报 → 所有类型都有一定需求，竞品几乎没做
  🔵 听前预览 → 小宇宙简介过于简单，结构化预览有空间
```

## 二、跨类型共性需求（优先级排序）

| 需求 | 描述 | 当前状态 | 竞品覆盖 |
|------|------|----------|----------|

| 需求 | 描述 | 当前状态 | 竞品覆盖 |

### P2 — 知识型用户的深度功能

| 需求 | 描述 | 当前状态 | 竞品覆盖 |

## 三、类型特有的差异化需求

### 3.1 知识/教育型 — 当前主力用户

**专属需求**：
- 知识卡片（概念名 + 定义 + 例子 + 关联概念）→ **已有，是差异化优势**
- 方法论/框架类图表（flow/layers）→ **已有**
- 行动清单（从播客内容提取 actionable takeaways）→ **缺失，建议新增**

**当前适配度**：★★★★☆（4/5）— 功能丰富，缺听前预览和导出

### 3.2 深度访谈型 — 小宇宙核心品类

**专属需求**：
- 嘉宾信息卡（头衔 + 代表作 + 本期角色）→ **有 participants，但呈现不够突出**
- 对话脉络图（谁在什么时候说了什么核心观点）→ **缺失**
- 多角度观点对比（嘉宾A vs 嘉宾B 的立场差异）→ **有 comparison 图表，未与说话人绑定**

**当前适配度**：★★★☆☆（3/5）— 结构化能力强，但缺乏「对话感」

### 3.3 故事/叙事型 — 需要最谨慎的类型

**专属需求**：
- **听后模式**：默认隐藏剧透内容，听完后解锁 → **缺失，核心需求**
- 情节时间线（事件A → B → C 的故事弧线）→ **有 detailed_timeline，但非故事导向**
- 人物关系图 → **缺失**
- 情感标注（这段令人感动/震惊/沉思）→ **缺失**

**当前适配度**：★★☆☆☆（2/5）— 当前模板偏知识提取，对叙事型不友好

### 3.4 新闻/时事型 — 时效性挑战

**专属需求**：
- 快速生成（< 30分钟内完成可视化）→ **当前 3-5 分钟，可接受**
- 事件脉络时间线（不是播客结构，而是事件本身的时间线）→ **缺失**
- 多方观点并列 → **有 arguments，但未按立场分组**

**当前适配度**：★★★☆☆（3/5）— 功能适配但缺事件视角

### 3.5 闲聊/生活型 — 可视化价值最低

**专属需求**：
- 轻量化呈现（不要长篇大论的分析，金句 + 氛围就够了）→ **缺失轻量模式**
- 社交分享优先（好看的海报 > 深度笔记）→ **缺失**
- 有趣片段标记（"这段笑死了"级别的高光时刻）→ **缺失**

**当前适配度**：★☆☆☆☆（1/5）— 当前产品对这类播客过度分析，不如不做

## 四、需求优先级总结

### 第一梯队：立竿见影（覆盖广 + 差异化强）

1. **听前预览模块** — 30秒决策，覆盖所有类型
2. **社交分享卡片/海报** — 蓝海机会，闲聊/访谈型的增长引擎
3. **播客类型自适应** — 根据内容类型调整模板重点（而非一刀切）

### 第二梯队：深度强化（留住核心用户）

4. **知识导出（Markdown）** — 知识型用户的刚需，提升粘性
5. **金句导出功能** — 已有收藏 UI，只差最后一步
6. **对话流视图** — 访谈型的差异化呈现

### 第三梯队：拓展边界（新用户类型）

7. **故事型专属模板** — 情节时间线 + 防剧透模式
8. **轻量模式** — 闲聊型的极简呈现（只留金句和分享）
9. **原文时间戳跳转** — 技术依赖小宇宙 deeplink

## 五、用户画像 × 产品定位

### 核心目标用户（占小宇宙用户 60%+）

```
25-35岁 | 一二线城市 | 本科以上 | 知识型+访谈型播客听众
通勤/运动场景听播客 → 事后想回顾要点 → 发现可视化总结
核心价值：效率（快速回顾） + 质量（呈现品质高于笔记）
```

### 增长目标用户

```
社交分享型用户：听到好播客想推荐给朋友 → 需要精美的分享素材
品牌播客运营：企业自有播客 → 需要每集的可视化传播物料
```

### 当前不应重点投入的用户

```
闲聊型听众：可视化价值低，强行服务会稀释产品定位
纯PKM用户：Snipd/Podwise 已足够好，不值得正面竞争
```
