递归大师 — OpenClaw AI 运营大脑
演讲稿（共16页，约10分钟）
========================================


【第1页 · 封面】

大家好，我是XX。今天给大家看一只不一样的龙虾——我们叫它"递归大师"。

为什么叫递归？因为它的核心机制就是递归：使用 → 犯错 → 记录 → 学习 → 使用得更好 → 犯更少的错 → 记录更精准的规则……每一轮循环，它都比上一轮更强。这不是线性进步，是递归式进化。


【第2页 · 我为什么来养虾】

先说实话：在做这个项目之前，我其实不太想用 OpenClaw。

我一直用 Claude Code，觉得已经够用了——对话式编程、上下文理解、代码生成都很强，没什么动力再折腾一个新工具。

但后来我仔细对比了一下，发现它们各有所长：

Claude Code 强在哪？
- 深度代码理解——能读懂整个项目架构、文件依赖、git 历史
- 精准的上下文——跨几十个文件重构，上下文不会丢
- 开发者体验好——终端里就能完成复杂的工程任务

OpenClaw 强在哪？
- 后台常驻 + 定时任务——cron 机制让它凌晨3点也能帮你跑数据、发报告
- 消息平台原生——团队成员通过微信、Telegram、Slack 都能跟同一只虾对话
- 主动执行——不用等人下指令，它可以自己监控、自己触发工作流

一个是最聪明的开发搭档，一个是最勤快的自动化助手。

我想到一个可能性：如果把 Claude Code 的"深度理解能力"注入到 OpenClaw 的"自动化执行框架"里呢？

这就是这个项目的出发点——不是二选一，而是把两者的优势整合在一起。


【第3页 · 真正的问题】

（语气平和，不夸张。）

先承认一件事：OpenClaw 本身已经不笨了。它有记忆，有人格设定，能跨会话记住东西。

但在实际团队使用中，我发现了三个真实痛点：

1. 记忆是散的 — MEMORY.md 靠人手动维护，没有结构，容易遗漏。纠正过的错误，可能下个月换个问法又犯了。

2. 每人一只虾，经验不互通 — 我教会了我的龙虾怎么分析外卖数据，但同事的龙虾完全不知道。团队三个人，各养各的，知识是三份孤岛。

3. 不会自我改进 — 它很勤奋地干活，但不会回头看"我是不是在重复做同样的事"，不会主动把经验沉淀成更高效的工作方式。

注意，这不是说龙虾蠢。这是说：一个聪明的个体，和一个有学习体系的团队成员，是两回事。


【第4页 · 个人工具 vs 团队大脑】

普通 OpenClaw：一个聪明的个人助手。能记住你告诉它的事，但记忆没有结构，也不会传播。
我们的 OpenClaw：一个有学习体系的团队成员。会结构化地记录教训，会自动在团队间同步，会主动从重复中进化。

一句话：我们给龙虾装了"错误免疫系统"和"团队经验网络"。


【第5页 · 过渡】

说再多不如看一次。5个场景，同样的问题，看看差别在哪。


【第6页 · 场景一：结构化的错误记忆】

问题：我们现在有多少家门店？

左边：龙虾凭记忆回答了12家。其实它的 MEMORY.md 里可能有正确数字，但没有一个机制提醒它"这个问题上次答错过"。

右边：我们的龙虾在回答前，自动检查了错题本。发现这个问题上周就命中过一次错误记录，于是它不猜了，直接去查数据库，返回准确的8家。

区别不是"能不能记住"，而是"有没有一个结构化的自检机制"。


【第7页 · 场景二：公司特有的经验规则】

这个场景特别贴近日常工作。

Day 1，龙虾说"转化率约15%"。用户纠正：要排除试喝券，正确是11%。

这种规则每个公司都有——口径怎么对齐、哪些数据要排除、时区怎么转换。不是能力问题，是经验问题。老手闭着眼睛都知道，新接手的人一定会栽。

普通龙虾：用户纠正了，它当时知道了。但8天后再问，它可能又忘了——除非用户手动更新了记忆文件。

我们的龙虾：被纠正的瞬间，自动追加一条结构化记录。8天后再问，系统先加载这条规则，直接给出"已排除试喝券：11.2%"。

这类公司特有的经验规则，一旦被记住就不会再丢。


【第8页 · 场景三：团队知识共享（核心差异）】

这是我们最独特的能力，也是普通 OpenClaw 完全做不到的事。

三个团队成员，都在分析 UberEats 渠道：
- 成员A：UberEats 单量下滑怎么分析？
- 成员B：UberEats 复购率怎么算？
- 成员C：UberEats 客单价为什么比到店高？

普通龙虾：三个人的龙虾各自独立回答，质量参差不齐。A 花了半天摸索出一套分析框架，B 和 C 完全享受不到。

我们的龙虾：
- 成员A 研究完，龙虾自动记录并同步到团队。
- 成员B 提问时，系统加载了 A 的研究成果，在此基础上精准回答。
- 成员C 提问时，综合了 A 和 B 的积累，给出更深入的分析。

一个人学一次，三个人都受益。这不是记忆力的问题，这是知识网络的问题。


【第9页 · 场景四：严谨性——不确定就追问】

有人问龙虾："这个月的营收增长了多少？"

左边的龙虾直接回答："增长了约23%，主要受益于新门店开业和季节性需求增长。"听起来很专业，但仔细看——"这个月"是 MTD 还是完整月？"营收"是总实收还是饮品实收？"增长"是环比还是同比？全都没说清楚。

右边的龙虾不一样。它触发了严谨性检查，发现三个口径没有对齐，于是反过来追问：
- 时间范围：MTD 还是等月底？
- 营收口径：总实收、饮品实收、还是含外卖？
- 对比基准：环比还是同比？

它说："请确认后我来查数据。我不会在口径不明确时给出数字。"

这个能力的价值在于：它不会为了显得有用而编一个模糊的答案。在数据分析里，一个口径不清的数字比没有数字更危险。


【第10页 · 场景五：从重复劳动到自动进化】

连做三周周报，会发生什么？

普通龙虾：每次都认真做，但不会意识到"我在重复"。第三周的质量不比第一周好多少。

我们的龙虾：
- Week 1 做报告，同时标记"这是一个重复模式"
- Week 2 再做，检测到第2次，主动优化输出结构
- Week 3 第3次命中，主动提议："这个分析你已经让我做了3次，要不要沉淀成一个固定技能？以后一句话就能出结果。"

它不只是在干活，它在观察自己的工作模式，主动提出优化建议。


【第11页 · 我们加了四样东西】

总结一下，我们在普通 OpenClaw 的基础上加了四样东西：

1. 结构化错题本（.learnings/）— 被纠正时自动记录，回答前自动自检
2. 知识晋升通道 — 出现1次留在笔记，2次升入长期记忆，3次写入核心规则
3. 团队知识网络（Git同步）— 一个人的学习，全团队受益
4. 模式识别与自进化 — 发现重复劳动，主动提议沉淀为技能

前两个是让单只龙虾更可靠，后两个是让龙虾从个人工具变成团队基础设施。


【第12页 · 记忆架构对比】

普通 OpenClaw 有2层记忆：会话记忆 + MEMORY.md。能记住东西，但记忆不会反向影响行为。

我们扩展到5层，关键不是层数多，而是加了两个机制：
- 自动晋升通道：错误记录从 .learnings/ → MEMORY.md → SOUL.md，最终改变龙虾的回答方式
- 团队同步层：通过 Git，让知识在成员间流动

记忆不只是"存着备查"，而是"会改变行为"。这是本质区别。


【第13页 · 进化飞轮】

使用 → 纠正/学习 → 自动记录 → 团队同步 → 模式识别 → 知识晋升 → 更可靠 → 使用...

第1周：积累5条经验，开始建立团队知识库
第1月：23条结构化规则 + 3个自动技能，新成员加入直接继承
第3月：这只龙虾变成了团队的经验合集，新人入职可以直接受益

飞轮的核心动力不是个人使用，是团队协作——人越多，知识积累越快，每个人受益越大。


【第14页 · 算笔账】

3人 × 30分钟/天 × 365天 = 540小时/年。这是省下的重复劳动。

但更直接的价值是三件事：
- 新人入职：龙虾里有全团队积累的经验规则，上手更快
- 换岗协作：接手新业务线，前任的经验都在龙虾里
- 日常工作：口径对齐、操作规范这些经验类的东西，系统会自动提醒


【第15页 · 金句收尾】

（放慢语速。）

我一开始只是想把 Claude Code 的深度理解能力和 OpenClaw 的自动化框架整合在一起。

做着做着发现，真正有价值的不是让一只龙虾更聪明——而是让整个团队的经验，变成一张不会断裂的网。

别人的 OpenClaw 是一个聪明的个人助手。
我们的 OpenClaw 是一个团队共享的运营大脑。
