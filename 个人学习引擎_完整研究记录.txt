第一章：100LS 英语学习法
================================================================================

一、起源与作者
──────────────

来源：韩国作者张同完（장동완）所著《我在100天内自学英文翻转人生》。
他声称该方法灵感源自美国国防部语言学院（DLI）——二战期间为快速培训
士兵掌握敌方语言，每天高强度训练20小时，6个月后士兵能像母语者一样
流利沟通。

作者本人成果：用此方法1年内达到英文口译水平，随后又学会了日语、
法语和中文，20多岁就获得了200万年薪的工作邀约。


二、核心理念
────────────

100 = 100遍 & 100天
L = Listening（听）
S = Speaking（说）

核心哲学：语言是运动技能，不是学科知识——跟学钢琴、学游泳一样，
靠的是肌肉记忆和反复训练，不是"理解语法"。

学习顺序的颠覆：
  传统方式：阅读（语法）→ 写作 → 听力 → 口语
  100LS方式：听力 → 口语 → 阅读（语法）→ 写作

这与婴儿学母语的路径一致：先听父母说 → 模仿开口 → 再学认字和写字。


三、具体七步操作法
──────────────────

  步骤        操作                                目的
  ──────────  ──────────────────────────────────  ──────────────────────────
  第1遍       关掉所有字幕完整看一遍              感知整体语境，建立听觉印象
  第2遍       打开中文字幕再看一遍                理解剧情，确认听不懂的部分
  第3遍       切换英文字幕，抄写不懂片段，查生词  扫除词汇障碍
  第4-N遍     逐段反复听+跟读，模仿语调/语速/情感 练到完全跟上演员语速
  剩余至100   关掉所有字幕，听+跟读               内化语感，形成条件反射
  第6步       参加英语交流聚会，带笔记实战        输出验证
  第7步       用同样方法再完成9部电影（共10部）    扩展语境覆盖

时间规划：1部电影 ≈ 100天，10部电影 ≈ 1000天（约3年）


四、选材原则
────────────

- 强烈推荐现代爱情片：对话贴近日常生活
- 避免：科幻片、战争片（台词脱离日常）
- 避免：老情景喜剧如《老友记》（语言过时）
- 作者推荐片单：《诺丁山》《猫鼠游戏》《真爱至上》《BJ单身日记》
- 进阶路径：10部电影完成后 → TED演讲 → 英文新闻


五、背后的科学原理
──────────────────

1. Krashen的输入假说（Input Hypothesis）
   - 语言习得发生在接收到"可理解性输入"（i+1）时
   - 前3遍字幕辅助 = 把不可理解变成可理解
   - 后97遍无字幕 = 巩固已理解的内容

2. 程序性记忆（Procedural Memory）
   - 语言不是"知道"而是"会做"
   - 需要大量重复形成自动化神经通路
   - 训练口腔肌肉记忆 + 听觉模式识别

3. 神经元突触强化（Hebbian Learning）
   - "Neurons that fire together, wire together"
   - 反复听 → 声音→语义→语境的神经连接越来越强
   - 最终"不翻译就直接理解"

4. 情境学习（Situated Learning）
   - 电影 = 真实语境（表情、肢体、场景、情感）
   - 语言与情境绑定 → 记忆更牢固

5. 窄输入假说（Narrow Input）
   - 同一电影反复 = "窄输入"策略
   - 有限材料里高频曝光 → 加速内化


六、客观评价
────────────

优势：方法简单、执行门槛低、符合语言习得规律、针对哑巴英语
局限：100遍枯燥、鸡汤偏多、缺系统词汇扩展、覆盖面有限
本质：把核心原理（大量可理解输入+重复+模仿输出）包装成极简框架

第二章：克拉申（Stephen Krashen）第二语言习得理论
================================================================================

一、人物背景
────────────

Stephen Krashen，美国南加州大学（USC）语言学荣誉退休教授，
二语习得领域最具影响力的学者之一。
1980年代初提出"监控模型"（Monitor Model），由六大假说构成。


二、六大核心假说
────────────────

【假说1：习得-学习假说（Acquisition-Learning Hypothesis）】

根基性区分：

  习得（Acquisition）                学习（Learning）
  ──────────────────────            ──────────────────────
  无意识，自然发生                    有意识，刻意学习
  类似儿童学母语                      课堂学语法规则
  产生语感、直觉判断对错              知道规则但不一定会用
  能流利输出                          只能"监控"输出

核心观点：真正让人流利使用语言的是"习得"而非"学习"。
课堂语法规则只能做编辑/监控，不能让你自然开口。


【假说2：监控假说（Monitor Hypothesis）】

"学习"到的知识只在三个条件同时满足时才能发挥作用：
1. 有足够时间思考
2. 注意力集中在语言形式上（而非意义）
3. 知道相关规则

日常对话中这三个条件几乎不可能同时满足，
所以语法知识在实际交流中作用有限。


【假说3：自然顺序假说（Natural Order Hypothesis）】

语法结构的习得有固定先后顺序，不因母语、教学顺序而改变。

英语习得的大致顺序：
  进行时(-ing) → 复数(-s) → 系动词(be) → 助动词 → 冠词(a/the)
  → 不规则过去式 → 规则过去式(-ed) → 第三人称单数(-s)

启示：按教材顺序教语法可能无效，大脑有自己的"内置课程表"。


【假说4：输入假说（Input Hypothesis）⭐ 最核心】

公式：i + 1

  i = 学习者当前语言水平
  +1 = 略高于当前水平的新内容
  语言习得只在接收到"可理解性输入"时才发生

四个关键条件：
  1. 可理解性：听不懂的输入 = 噪音，无效
  2. 足够的量：需要大量、持续的输入
  3. 关注意义而非形式：注意力在"说了什么"而非"怎么说的"
  4. 趣味性/相关性：内容要让学习者感兴趣

颠覆性主张：
  说话（输出）不是习得的原因，而是习得的结果。
  不需要"练习说"来学会说——输入够了说话能力自然涌现。


【假说5：情感过滤假说（Affective Filter Hypothesis）】

即使有完美的i+1输入，情感过滤器"开高了"→ 输入无法被吸收：

  可理解输入 → [情感过滤器] → 语言习得装置 → 习得
                   ↑
            焦虑/紧张/自卑/被迫 = 过滤器升高 = 输入被阻断

低过滤器状态：
  - 内在动机强（"我真的想学"而非"被逼着学"）
  - 自信心（"听不懂也没关系，继续听"）
  - 低焦虑（放松、享受、没有被评判的压力）


【假说6：令人着迷的输入假说（Compelling Input）🆕】

克拉申近年补充：输入不仅要"可理解"，还要"compelling"（令人着迷）。

  "有趣"：觉得这个内容不错
  "着迷"：欲罢不能，不自觉地继续听/看下去

当输入达到compelling级别时，情感过滤器自动降到最低，
学习者甚至意识不到自己在学外语。


三、克拉申的三种实践方法
────────────────────────

方法A：自由自主阅读（Free Voluntary Reading, FVR）
  克拉申称之为"语言教育中最强大的工具"
  原则：读自己想读的内容，不查字典，没有任务/测试/压力
  数据：FVR的词汇习得效率是显性教学的2-6倍

方法B：窄读（Narrow Reading）
  深入阅读同一作者/同一话题的大量材料
  原理：同一作者词汇句式高度重复 → 自动复习
  例：读完一本John Grisham → 下一本自动变简单

方法C：窄听（Narrow Listening）
  收集同一话题的多段母语者录音，反复听
  操作：选话题 → 找3-5段不同人讲 → 反复听 → 换话题
  100LS就是窄听的极端版本


四、对克拉申的批评与补充
────────────────────────

【Swain的输出假说】

加拿大法语沉浸式教学（1965起，近20年）结果：
  ✅ 听力和阅读优秀
  ❌ 口语表达和语法准确性很差

结论：光有输入不够，还需要"被推动的输出"（Pushed Output）

输出的三大功能：
  1. 注意功能：说/写时才发现"这个我不会表达"
  2. 假设检验：说出来 → 对方反应告诉你对不对
  3. 元语言反思：用语言思考语言本身

【2025年神经科学批评】

Frontiers in Psychology（2025）指出：
  - 语言习得是主动的、具身的（embodied）过程
  - 依赖互动和反馈，不是被动接收
  - 但同期AI研究验证了分级阅读材料确实有效

综合评价：输入假说不是错的，而是不完整的。
最佳路径 = 大量可理解输入 + 适时推动输出 + 互动反馈


五、60-70%理解度的科学依据
──────────────────────────

这个比例对应Vygotsky的最近发展区（ZPD）：

  < 50% 理解：挫败区——太难，放弃或焦虑
  60-70% 理解：发展区——有挑战但可理解，大脑主动填补空白 = 习得发生
  80-90% 理解：舒适区——太容易，缺少+1
  95%+ 理解：巩固区——适合复习


六、泛化训练的重要性
────────────────────

克拉申的盲点：只讨论"理解意义"，但现实听力挑战还包括：
  - 语速变化：新闻播报 vs 日常闲聊 vs 快速争论
  - 口音差异：美式 vs 英式 vs 印度 vs 澳洲
  - 语音现象：连读、弱读、吞音、缩略
  - 背景噪音：真实环境永远不是录音棚品质

泛化方案：80%核心材料（窄听/窄读）+ 20%泛化材料

第三章：值得研究的实践案例
================================================================================

【案例1：Steve Kaufmann — 20门语言的多语者】

  身份：加拿大人，LingQ平台创始人，YouTube语言学习博主
  语言数：20门语言
  核心方法：大量听 + 大量读（comprehensible input信徒）
  关键习惯：学新语言时，同一个mini story听约40遍
  工具：自创LingQ平台（阅读+标记生词+听力一体化）
  克拉申评价：克拉申研究过他的案例，认为是理论的独立验证

  核心理念：
  "The best way to learn a language is to massively ingest it,
   by listening and reading."

  六大要素：
  1. 态度（相信自己能学会）
  2. 时间（每天至少1小时）
  3. 注意力（关注新出现的词和模式）
  4. 听和读（占80%的学习时间）
  5. 词汇（通过上下文积累，不靠背单词表）
  6. 享受过程（选自己感兴趣的内容）


【案例2：Kató Lomb — 自学16门语言的传奇】

  身份：匈牙利人，世界上最早的同声传译员之一
  背景：化学和物理学博士，成年后才开始学语言
  语言数：16门语言（含中文、日文、俄语等）
  核心方法："核心小说法"——用一本小说作为入门材料反复阅读
  公式：投入时间 + 兴趣 = 结果（没有捷径）
  时间投入：每周10-12小时
  著作：《Polyglot: How I Learn Languages》（1970）

  关键洞察：
  阅读是基石，通过读小说同时获得语法、词汇和语感。
  先读懂，再开口。


【案例3：AJATT / Matt vs Japan — 极端沉浸式】

  创始人：Khatzumoto（AJATT），后发展为 MIA / Refold
  方法：All Japanese All The Time — 所有时间沉浸在目标语言
  成果：声称在美国18个月内达到日语高级水平
  核心原则：不学语言，活在语言里；大量被动听+主动看
  输出时机：等到"感觉自然"才开口（可能几个月到一年以上）

  Refold四阶段：
  Stage 0：准备（基础发音 + 500核心词）
  Stage 1：建立理解力（大量输入）
  Stage 2：深化理解力（密集沉浸，每天3+小时）
  Stage 3：开始输出


【案例4：少数派作者的沉浸式实践】

  时间分配：单词20% + 语法20% + 沉浸学习60%
  选材三原则：感兴趣 + 生词率≤10% + 适度挑战
  工具：Anki（间隔重复）+ 句子卡片（一卡一个生词）
  独特做法：用工作相关英文产品手册（一致性效应）
  科学依据：内在动机 + 间隔重复 + 一致性效应 + 信息冗余

第四章：Bloom 2 Sigma 问题
================================================================================

一、研究背景
────────────

1984年，芝加哥大学教育心理学家Benjamin Bloom在《Educational Researcher》
发表论文：接受一对一辅导+掌握学习法的学生，平均成绩比传统课堂学生
高出2个标准差（2σ）——普通辅导学生超过传统课堂中98%的学生。


二、实验设计
────────────

  条件            师生比    教学方式                      效果
  ──────────────  ────────  ──────────────────────────    ──────
  传统课堂        1:30      统一进度、统一考试            基准（0σ）
  掌握学习        1:30      加反馈-纠正循环               +1σ
  一对一辅导      1:1~1:3   个性化节奏+掌握学习           +2σ


三、掌握学习法（Mastery Learning）的具体机制
────────────────────────────────────────────

  学习单元 → 形成性测试 → 达标？
                            │
                    ┌───────┴───────┐
                    ↓ 是            ↓ 否
                拓展活动        纠正活动
                    │              │
                    │         再次测试
                    │              │
                    └──────┬───────┘
                           ↓
                     下一个单元

六个关键步骤：
  1. 明确学习目标：掌握标准通常80-90%正确率
  2. 组织为小单元：每个单元1-2周内容量
  3. 高质量初始教学：先学核心内容
  4. 形成性测试：诊断——精确识别"哪里会了，哪里没会"
  5. 针对性纠正：只补不会的部分
  6. 二次测试验证：确认真的掌握了


四、一对一辅导为什么额外多出1σ？
──────────────────────────────────

  维度          课堂掌握学习              一对一辅导
  ────────────  ────────────────────────  ────────────────────────
  节奏          全班统一进度              完全适配个人节奏
  诊断精度      通过测试发现问题          实时观察+即时发现
  反馈延迟      测试后才知道              说错的瞬间就纠正
  情感支持      有限                      持续关注、鼓励、降低焦虑
  注意力        分散在30人                100%聚焦于你
  材料适配      统一教材                  根据水平实时调整难度


五、后续研究的修正（诚实面对数据）
──────────────────────────────────

Bloom的2σ被后续研究认为有所夸大，但核心结论成立：

  方法            Bloom声称    实际验证            来源
  ──────────────  ──────────   ───────────────    ─────────────
  掌握学习        +1.0σ        +0.2~0.5σ          Slavin/Kulik元分析
  一对一辅导      +2.0σ        +0.5~1.0σ          VanLehn综述(d=0.79)
  高质量AI辅导    —            +0.5~1.0σ(极端2~3σ) DARPA Digital Tutor

Slavin关键发现：
  掌握学习效果在控制学习时间后大幅缩减——
  说明部分效果来自"多花了时间"而非方法本身。

真正有效的底层机制：
  1. 频繁测试（testing effect）：测试本身增强记忆
  2. 即时反馈：知道对错比"继续练"更重要
  3. 精准纠正：只补短板，不浪费时间在已会的内容上
  4. 适配难度：材料匹配学习者水平

2025-2026 AI应用：
  Khan Academy的Khanmigo实验：AI辅导实现2倍学业增长
  DARPA Digital Tutor：效果量达d=1.97~3.18（超过Bloom原始数据）


六、与Ericsson刻意练习的关系
────────────────────────────

  Bloom掌握学习                  Ericsson刻意练习
  ────────────────────────────   ────────────────────────────
  明确学习目标                   明确的表现目标
  形成性测试诊断                 持续的表现评估
  针对性纠正                     聚焦弱点练习
  达到掌握标准才进阶             在舒适区边缘练习
  导师/辅导者指导                专家教练指导

核心共识：不是练习量决定水平，而是练习的质量和精确度。

第五章：三大理论融合框架（三层引擎模型）
================================================================================

一、融合逻辑
────────────

  克拉申 → 解决"学什么"和"怎么输入"
  Bloom  → 解决"怎么确保真的学会了"
  泛化   → 解决"怎么用到真实世界"

  输入 → 吸收 → 检验 → 纠正 → 掌握 → 泛化 → 实战
   ↑       ↑       ↑       ↑       ↑       ↑       ↑
  克拉申  克拉申   Bloom   Bloom   Bloom   泛化    Swain
  i+1   情感过滤  测试    纠正    标准    训练    输出


二、三层引擎模型
────────────────

【层1：输入引擎（克拉申）】

  参数          设定                         理论依据
  ────────────  ─────────────────────────    ──────────────
  理解度        听力60-70%，阅读90%+         i+1 + ZPD
  内容要求      Compelling（着迷级）          第六假说
  输入策略      窄听窄读80% + 泛化20%        Narrow Input
  心理状态      低焦虑、高动机               情感过滤
  注意力        关注意义，不分析语法         习得-学习假说

【层2：掌握引擎（Bloom）】

  学习单元
      ↓
  第一轮输入：听/读（i+1）
      ↓
  形成性自测（诊断）
    ├── 能听懂多少？（百分比自评）
    ├── 能复述大意吗？（理解检验）
    ├── 哪些词/句听不懂？（定位短板）
    └── 能跟读到什么程度？（输出检验）
      ↓
  针对性纠正
    ├── 查不懂的词/短语
    ├── 反复听不清的片段
    ├── 模仿跟读直到流畅
    └── 只练不会的部分
      ↓
  二次验证
    ├── 关掉参考，能听懂90%+了吗？
    ├── 能自然跟读/复述了吗？
    └── 是→下一单元 / 否→再纠正

关键洞察：
  100LS的"听100遍"是粗放的掌握学习。
  Bloom框架让你精准定位短板、靶向纠正，
  可能20-30遍就能达到100遍的效果。

【层3：泛化引擎】

  阶段      核心活动                  标准                  泛化拓展
  ────────  ────────────────────────  ──────────────────    ──────────────
  精学期    一个材料精听到掌握        理解度90%+            暂无
  泛化期    同话题换不同来源          每个来源70%+          多语速/口音
  实战期    真实场景对话              能理解并回应          噪音/多人/即兴


三、100LS vs 融合模型对比
─────────────────────────

  维度          100LS原版              融合模型
  ────────────  ──────────────────    ──────────────────────────
  输入策略      一部电影听100遍       窄听为核心，60-70%选材
  诊断机制      ❌ 无（靠遍数堆）      ✅ 每轮后自测诊断
  纠正精度      粗放（整部重复）      ✅ 只练不会的片段
  掌握标准      100遍=完成            ✅ 90%+理解+能复述=完成
  泛化训练      ❌ 无                  ✅ 同话题多来源
  输出训练      跟读                  ✅ 跟读→复述→对话
  个性化        ❌ 统一流程            ✅ 根据自测调整
  效率          100遍                 可能20-30遍同效果

第六章：专家级评价（对三层引擎模型的批判性审视）
================================================================================

一、做对了什么
──────────────

1. 理论融合的角度正确。把克拉申（输入）、Bloom（验证）、泛化（迁移）
   接在一条链上，市面上少见，逻辑自洽。

2. 60-70%理解度的量化锚点实用，比"略高于你的水平"具体得多。

3. 用诊断-纠正循环替代"盲目听100遍"，方向完全正确。


二、五个结构性问题
──────────────────

【问题1：克拉申和Bloom之间有根本性哲学冲突】

  克拉申：习得是无意识的，关注意义而非形式。
  Bloom：学习要有意识地诊断、纠正、验证。

  这两个指令是矛盾的。

  当你听完播客后"自测：能听懂多少？哪些词不会？"——
  这个动作把你从习得模式（无意识吸收）切换到学习模式（有意识分析）。
  按克拉申的理论，这恰恰会激活"监控器"，拉高情感过滤器。

  现实表现：你本来沉浸在播客里（compelling input，过滤器低），
  听完做"诊断测试"——列不会的词、评估理解百分比——
  你从"享受内容的人"变成了"评判自己的学生"。焦虑感回来了。

  建议："复述"可能比"自测打分"更好——
  复述既检验了理解，又不会让人进入自我评判模式。


【问题2：模型过度理论化，执行复杂度可能反噬学习本身】

  成功的多语者案例共同特征：执行时的认知负荷极低。
  - Kaufmann：听+读，标记生词，重复。就这么简单。
  - Lomb：找一本小说，反复读。就这么简单。
  - AJATT：把所有时间泡在目标语言里。就这么简单。

  三层引擎有14个步骤、3个引擎、4个参数维度……
  作为理论框架完整，但作为每日执行方案太重了。

  认知科学的反讽：你在学习"如何学英语"上花的认知资源越多，
  留给"实际学英语"的就越少。这叫元认知过载。

  建议：框架需要"极简执行版"——
  日常执行只记2-3个规则，复杂框架只在月度复盘时拿出来。


【问题3：最关键的变量被当成参数而非引擎——坚持】

  所有语言学习失败案例的第一死因不是方法错误，而是放弃了。
  100LS最大挑战："同一电影听100遍太无聊，第30遍就放弃了"
  Refold最常见帖子：不是"方法不work"，是"坚持3个月就断了"

  模型把动机/情感处理为"参数"，但应该是独立的"持续引擎"：
  - 习惯形成：怎么让每天听英语变成像刷牙一样的自动行为？
  - 身份认同：从"我在学英语"变成"我是使用英语的人"
  - 环境设计：手机改英文、播客首页全英文、社交关注英语账号
  - 社交绑定：学习伙伴、社群、定期有人跟你说英语
  - 进步可见性：怎么让自己感受到进步
  - 最小行动阈值：状态不好做5分钟也算数

  注意：你的LifeOS其实已经是这个"持续引擎"——
  这是你比大多数学习者领先一大步的地方。


【问题4：泛化层最有原创性，但也最不成熟】

  缺乏：
  - 具体怎么做？怎么找同话题但不同口音/语速的材料？
  - 怎么衡量泛化成功？理解70%怎么测？
  - 泛化的顺序？先变语速还是先变口音？
  - 与精学阶段的切换时机？

  建议：从"概念"升级为"可执行的操作手册"。


【问题5：缺少记忆科学这一环】

  讨论了输入、验证、迁移，但完全没有讨论遗忘。

  Ebbinghaus遗忘曲线：
  - 学完24小时后遗忘约70%
  - 1周后遗忘约80%

  所有成功案例里（Refold、少数派、Kaufmann的LingQ），
  SRS（间隔重复）都扮演了关键角色。

  但模型里"复习"完全没有位置。
  Bloom循环是"学-测-纠-过"，过了就下一个单元了。
  但语言不像数学——你会忘记上周学的单词。

  建议：在掌握引擎和泛化引擎之间加入"记忆维护"机制。

  ─────────────────  ──────  ─────────────────────────────
  理论完整性          8/10    覆盖输入→验证→迁移，但缺记忆
  内部一致性          6/10    克拉申和Bloom有哲学冲突

四、升级建议：四层引擎 + 极简执行
─────────────────────────────────

  升级后架构：

  第0层：持续引擎（习惯 × 环境 × 社交 × 最小行动）
    ↓ 确保你每天都在做
  第1层：输入引擎（克拉申i+1，compelling，窄听窄读）
    ↓ 吃进去
  第1.5层：记忆维护（间隔重复SRS，对抗遗忘曲线）
    ↓ 留得住
  第2层：掌握引擎（Bloom式诊断-纠正，用"复述"替代"自测打分"）
    ↓ 确认真会了
  第3层：泛化引擎（多语速/口音/语境，需落地为操作手册）
    ↓ 在真实世界能用

  极简日常版（每天只记3条）：
  1. 听/读你着迷的英文内容（输入）
  2. 听不懂的地方重听+查+跟读（诊断纠正）
  3. 新词加入Anki，每天复习5分钟（记忆维护）

 The best method is the one you actually do every day."

【克拉申核心】
  习得 ≠ 学习 → 用耳朵和嘴巴，不是用笔和脑
  i + 1 → 理解度60-70%的材料最有效
  Compelling > Interesting > Boring → 着迷 > 有趣 > 无聊
  窄听/窄读 > 广泛涉猎 → 一个话题吃透再换
  输入优先，输出跟随 → 先听懂，后开口
  低焦虑 = 高吸收 → 放松状态下学习效率最高

【Bloom核心】
  形成性测试→诊断短板 → 不是打分，是找问题
  靶向纠正→只练不会的 → 不浪费时间重复已会的
  掌握标准→90%+才进阶 → 不追遍数，追质量
  即时反馈 > 延迟反馈 → 越快知道对错越好
  测试本身增强记忆 → testing effect

【泛化核心】
  多语速/多口音/多语境 → 锻炼实战能力
  精学80% + 泛化20% → 深度与广度的平衡

【持续引擎（评价中补充）】
  习惯 > 意志力 → 自动化行为
  环境设计 > 自我约束 → 改变环境而非自己
  最小行动 > 完美执行 → 5分钟也算数
  进步可见 > 目标宏大 → 感受到进步才能坚持
  身份认同 > 任务清单 → "我是使用英语的人"

【记忆维护（评价中补充）】
  遗忘曲线：24h忘70%, 1周忘80% → 必须间隔重复
  Anki/SRS → 最高效的对抗遗忘工具
  句子卡片 > 单词卡片 → 一卡一个生词+完整语境

【最终公式】
  有效习得 = 可理解输入(60-70%)
           × Compelling程度
           × 掌握验证循环(测试→纠正→再测试)
           × 泛化训练(多语速×多口音×多语境)
           × 低情感过滤(低焦虑×高动机)
           × 间隔重复(对抗遗忘)
           ÷ 无效重复(已会的不再练)

【输入输出阶段比例】
  初期(0-3月)  90:10  以听读为主，输出仅跟读/模仿
  中期(3-9月)  70:30  加入简单对话，写短文
  后期(9月+)   50:50  大量实战对话、写作、演讲

【极简日常版（每天只记3条）】
  1. 听/读你着迷的英文内容（输入）
  2. 听不懂的地方重听+查+跟读（诊断纠正）
  3. 新词加入Anki，每天复习5分钟（记忆维护）
