ARCHITECTURE

五层架构图

从灵魂到防线，每一层都是精心设计的智慧结构

🛡️ Layer 5 · 顶层

严谨层 Rigor Rules

  • 事实必须引用来源
  • 数字必须有单位和时间范围
  • 不确定时明确标注置信度
  • 每次输出后自检：有没有遗漏、矛盾
🤝 Layer 4

协作层 Git-synced Workspace

  • 3人共用一个 workspace（git 同步）
  • A 的学习 → B 和 C 立刻可用
  • 共享技能库：一人造技能，全团队受益
📚 Layer 3

学习层 .learnings/

  • 自动捕获：命令失败 → 记录 ERRORS.md
  • 用户纠正 → 记录 LEARNINGS.md
  • 缺失能力 → 记录 FEATURE_REQUESTS.md
  • 晋升机制：同类问题 3 次 → 自动晋升为永久规则
💾 Layer 2

记忆层 MEMORY.md + Daily Logs

  • 每日自动记录：做了什么决策、发现了什么
  • 长期精华：从日志中提炼高价值知识
  • 不是聊天记录，是结构化的运营知识库
🧠 Layer 1 · 底层

灵魂层 SOUL.md

  • 不是通用人格，是运营专家人格
  • 内置决策框架：面对模糊需求 → 先追问 → 列假设 → 确认
  • 领域知识：运营指标体系、分析方法论、常见陷阱

NEGATIVE FEEDBACK LOOP

负反馈闭环：犯过的错，永远不会再犯

普通龙虾被纠正后下次还会犯。我们的龙虾有完整的「错误免疫系统」——从发现错误到永久免疫，全自动闭环。

🚨

第一步：错误检测

三种触发方式，确保任何问题都不会漏掉：

用户纠正 "不对，应该是…" 自检矛盾 — 输出与已知事实冲突 命令失败 — 非零退出码 幻觉识别 — 编造了不存在的信息
📋

第二步：结构化记录到 .learnings/ERRORS.md

不是随便记个笔记。每条错误有唯一 ID、分类、根因分析、和防治规则：

ERR-20260317-001 类型: 幻觉 / 推理错误 / 过时知识 错误输出 vs 正确答案 根因分析 + 防治规则
🔍

第三步：模式识别

不是每个错误都要升级为规则。系统自动追踪关联错误——同类问题出现 2 次时预警，3 次时触发自动晋升。

See Also 交叉引用 30天滑动窗口 按领域聚类（运营/数据/技术）
⬆️

第四步：知识晋升（从临时记录 → 永久免疫）

防治规则沿着「信任阶梯」逐级晋升，最终写入 SOUL.md 成为龙虾的 DNA：

.learnings/ → 临时记录 MEMORY.md → 长期记忆 AGENTS.md → 操作硬规则 SOUL.md → 永久人格 DNA
🛡️

第五步：免疫验证

晋升后不是就完了。下次遇到同类场景时，系统会主动触发防御——在回答前检查是否命中已知错误模式，命中则走修正路径。

已知陷阱黑名单 回答前自动比对 命中则走修正路径 状态: pending → verified → promoted
⚡ 实战案例：幻觉免疫
